# ANIMATION_3D.md — Спецификация 3D-анимации

> Дополнение к `DESIGN.md` и `SKILLS.md`. Описывает, где именно на сайте используется
> 3D, какой библиотекой, и как обеспечить адаптивность и производительность на
> телефоне, планшете и ноутбуке. Все правила здесь обязательны к соблюдению —
> они не отменяют требование Lighthouse ≥85 из `WORKFLOW.md`.

## Общий принцип
3D используется **точечно**, не по всему сайту. Цель — произвести впечатление
профессионализма в первые секунды (Hero-секция), а не превратить портфолио в
3D-игру. Все остальные секции (Services, Portfolio, Contact) остаются на
обычной 2D/CSS-вёрстке из `DESIGN.md`.

## Где используется 3D

### 1. Hero-секция — главный 3D-акцент (версия 2)

**Проблемы версии 1 (исправить обязательно):**
- Октаэдр визуально перекрывал заголовок и подзаголовок — текст читался плохо.
- Сцена выглядела статично-скучной: одна фигура, вращающаяся в центре.

**Новая сцена — «Living Tech Constellation»:**
- **Композиция:** 3D-сцена размещается СПРАВА от текста на десктопе
  (layout: текст слева ~55% ширины, сцена справа ~45%), а не позади текста.
  На планшете — сцена меньше и смещена в правый верхний угол с пониженной
  прозрачностью. Текст ВСЕГДА поверх и полностью читаем — никаких элементов
  сцены под заголовком.
- **Центральный объект:** 3D-логотип "A" со скобками — экструдированная
  (объёмная) версия треугольного логотипа из `DESIGN.md`. Строится из
  сегментов-граней (как на визитке: часть граней синие `primary-bright`,
  часть светлые), материал с лёгким emissive-свечением. Медленное вращение
  вокруг вертикальной оси + плавное покачивание (sin по времени).
- **Окружение центрального объекта:**
  - Орбитальные кольца: 1–2 тонких наклонных кольца (torus с малым сечением),
    вращающиеся вокруг логотипа с разной скоростью — отсылка к circuit-эстетике
    визитки (рис. 3, где логотип в круге с узлами).
  - Частицы-точки: 80–150 маленьких светящихся точек (Points / buffer geometry,
    НЕ отдельные меши) — разбросаны сферически вокруг объекта, медленно дрейфуют.
    Цвет `glow`, размер 1–2px. Это дёшево по производительности и даёт «живость».
  - На десктопе несколько ближайших частиц слегка реагируют на курсор
    (отталкивание/притяжение в радиусе) — интерактивная «магия» без нагрузки.
- **Параллакс:** вся группа (логотип + кольца + частицы) плавно наклоняется
  за курсором на десктопе (lerp, как в версии 1). На планшете — только
  автовращение.
- **Свет:** ambient (слабый, холодный) + один directional/point с синим оттенком.
  Без теней. Emissive-материалы дают свечение без постобработки — bloom НЕ
  использовать (дорого).
- Цвета всех элементов — строго из палитры `DESIGN.md` (`primary`,
  `primary-bright`, `glow`, `ink` для светлых граней).

**Бюджет производительности сцены (жёсткий):**
- Суммарно ≤ 10 000 полигонов (экструзия логотипа low-poly, кольца с малой
  сегментацией: torus 64×8 максимум).
- Частицы — только Points с одной buffer-геометрией, не массив мешей.
- Один проход рендера, без постобработки, без теней, dpr [1, 1.5] — как раньше.
- Все правила версии 1 остаются в силе: lazy loading, мобильный fallback,
  prefers-reduced-motion, пауза при уходе из viewport, pointer-events: none.

### 2. Карточки проектов (Portfolio) — лёгкий 3D, не полноценная сцена
- Не WebGL, а CSS 3D-transform: лёгкий tilt-эффект карточки при наведении
  (`perspective` + `rotateX/rotateY` на пару градусов). Создаёт ощущение
  "объёма" без нагрузки на GPU.
- На тач-устройствах эффект наведения отключается (там всё равно нет hover) —
  карточка ведёт себя как обычная плоская карточка.

## Технологии
| Задача | Библиотека |
|---|---|
| 3D-сцена в Hero | `@react-three/fiber` + `@react-three/drei` (обёртка над Three.js под React) |
| Tilt-эффект карточек | Чистый CSS + небольшой JS-хук на mousemove, без библиотек |

Устанавливать: `npm install three @react-three/fiber @react-three/drei`

## Обязательные правила производительности

1. **Lazy loading.** 3D-сцена Hero грузится через `React.lazy()` + `<Suspense>`,
   не блокирует первую отрисовку остального сайта (LCP не должен зависеть от 3D).
2. **Мобильный fallback.** На экранах уже `md:` (< 768px) и/или при обнаружении
   слабого устройства — 3D-сцена заменяется на статичное SVG-изображение
   нового логотипа ("A" со скобками `<>`, сегментированные грани) с CSS-свечением
   (`shadow-glow`). Определять через `window.matchMedia('(max-width: 767px)')`
   и `navigator.hardwareConcurrency` (если ядер ≤ 4 — тоже статичная версия;
   если undefined — считать устройство мощным).
3. **Respect prefers-reduced-motion.** Если у пользователя включена настройка
   ОС "уменьшить анимацию" (`prefers-reduced-motion: reduce`) — всегда
   показывать статичную версию, независимо от устройства.
4. **Ограничение сложности сцены.** Не больше одной 3D-сцены на странице,
   не больше нескольких тысяч полигонов (low-poly стиль, не реалистичный рендер).
   Без теней (shadow maps), без постобработки (bloom, DOF) — это дорого для GPU.
5. **Пауза при не-видимости.** Если Hero-секция ушла из viewport при скролле —
   рендер-луп сцены должен приостанавливаться (не тратить GPU впустую).
6. **Проверка после реализации.** Обязательно прогнать Lighthouse на мобильном
   профиле именно на странице с 3D — если Performance < 85, сначала упрощать
   сцену/геометрию, и только потом (в крайнем случае) отказываться от 3D.

## Адаптивность по устройствам

| Устройство | Поведение |
|---|---|
| Ноутбук/десктоп (`lg:` 1024px+) | Полная 3D-сцена, параллакс за курсором |
| Планшет (`md:` 768–1023px) | Полная 3D-сцена, но без курсор-параллакса (автовращение) |
| Телефон (< 768px) | Статичное изображение того же мотива (без Three.js вообще — не загружать библиотеку) |

Важно: на телефоне библиотека `three`/`@react-three/fiber` не должна даже
попадать в загружаемый бандл этой страницы — использовать динамический импорт,
условный на медиа-запрос, а не просто скрывать 3D-канвас через CSS `display: none`
(иначе вес библиотеки всё равно скачается зря).

## Что НЕ делать
- Не добавлять 3D в Header, Footer, Services, TechStack — только Hero (+ лёгкий
  CSS-tilt в карточках Portfolio).
- Не использовать полноразмерные текстуры/HDRI-окружения — вся сцена должна
  оставаться на простых материалах и цветах из палитры бренда.
- Не блокировать скролл или взаимодействие с остальным сайтом ради 3D-эффекта.
