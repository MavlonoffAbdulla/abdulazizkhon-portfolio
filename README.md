# Portfolio — стартовый набор проекта

Этот набор файлов подготовлен для запуска разработки в **Antigravity** или **Claude Code**.
Здесь уже есть весь контекст, который агенту нужен, чтобы не переспрашивать про стек,
стиль и контент на каждом шагу.

## С чего начать (для агента)

Дай агенту такой первый промпт (или похожий):

> Прочитай PLAN.md, SKILLS.md, DESIGN.md, ANIMATION_3D.md, WORKFLOW.md и CONTENT.md в этом проекте.
> Начни с Этапа 0 из PLAN.md — инициализация проекта. После каждого шага отмечай
> прогресс в PLAN.md и следуй конвенциям из SKILLS.md, DESIGN.md и ANIMATION_3D.md.

## Что уже готово в этом наборе
| Файл | Назначение |
|---|---|
| `PLAN.md` | Пошаговый план — единственный источник правды по прогрессу |
| `SKILLS.md` | Стек, структура файлов, конвенции кода |
| `DESIGN.md` | Цвета, типографика, отступы, правила компонентов (фирменный стиль) |
| `WORKFLOW.md` | Git-конвенции, порядок добавления кейсов/языков, чеклист перед деплоем |
| `ANIMATION_3D.md` | Где используется 3D (Hero + tilt карточек), библиотеки, мобильный fallback, лимиты производительности |
| `CONTENT.md` | Черновые тексты всех секций + описания 7 кейсов портфолио |
| `src/locales/*.json` | Готовые переводы RU/UZ/EN (структура ключей синхронизирована) |
| `src/data/projects.js` | Данные кейсов, готовые к рендеру через `.map()` |
| `src/data/contacts.js` | Единая точка правды по контактам — **заполнить перед стартом** |
| `package.json`, `tailwind.config.js`, `vite.config.js`, `postcss.config.js` | Готовый конфиг сборки |
| `src/App.jsx`, `src/main.jsx`, `src/i18n.js`, `index.html` | Минимальный рабочий каркас (запускается сразу) |

## Перед тем как начать — заполни вручную
1. `src/data/contacts.js` — реальный Telegram, email, телефон.
2. `CONTENT.md` — все пометки `[уточнить у автора]` (результаты кейсов, стек MiniPOS Pro и т.д.).
3. Логотип (треугольная "A") — положить SVG/PNG в `src/assets/` (в наборе его нет, только описание в `DESIGN.md`).

## Быстрый старт локально
```bash
npm install
npm run dev
```
Откроется адрес из вывода Vite (обычно `http://localhost:5173`) — сейчас там просто
заглушка с текстом Hero-заголовка, что подтверждает: сборка, Tailwind и i18n уже работают.

## Порядок работы
Всё дальнейшее — по `PLAN.md`, начиная с Этапа 0. Не пропускай этапы и не меняй
порядок — план построен так, чтобы на каждом шаге был рабочий результат, который
можно посмотреть в браузере.
