# SKILLS.md — Технический контекст проекта

> Это "скилл"-файл для AI-агента (Antigravity / Claude Code). Он описывает стек,
> конвенции кода и структуру, чтобы агент генерировал код в едином стиле без
> необходимости повторно объяснять контекст в каждом промпте.

## Проект
Одностраничный (single-page) сайт-портфолио на React. Три языка (RU/UZ/EN).
Без backend — все данные статичные, обратная связь идёт через Telegram deep-link.

## Стек
| Слой | Технология |
|---|---|
| Сборка | Vite |
| UI | React (функциональные компоненты + hooks, без классов) |
| Стили | Tailwind CSS (только utility-классы, без отдельных .css кроме index.css с токенами) |
| Мультиязычность | react-i18next |
| Иконки | lucide-react |
| Анимации (2D) | Framer Motion (опционально, использовать по минимуму) |
| 3D-сцена (Hero) | `@react-three/fiber` + `@react-three/drei` — см. `ANIMATION_3D.md` |
| Хостинг | Vercel / Netlify (статика) |

> Правила использования 3D — обязательные, вынесены в отдельный файл
> `ANIMATION_3D.md` (lazy loading, мобильный fallback, ограничение сложности сцены,
> `prefers-reduced-motion`). Не начинать реализацию Hero-секции без прочтения этого файла.

## Конвенции кода
- Компоненты — функциональные, `export default function Name() {}`.
- Один компонент = один файл, имя файла = имя компонента.
- Никакого текста напрямую в JSX — весь текст идёт через `t('key')` из i18next.
- Ключи переводов — вложенная структура по секциям: `hero.title`, `services.items.erp.title` и т.д.
  Все три языковых файла (`ru.json`, `uz.json`, `en.json`) должны иметь идентичную структуру ключей.
- Категории кейсов — фиксированный список: `erp`, `bots`, `ai`, `automation`. Не придумывать новые
  категории без необходимости — это ломает фильтр в `Portfolio.jsx`.
- Все внешние ссылки (Telegram, mailto, tel) выносятся в constants в начале `Contact.jsx` или
  в отдельный `src/data/contacts.js`, не хардкодятся в нескольких местах.
- Цвета и отступы — только из токенов Tailwind-конфига (см. `DESIGN.md`), не использовать
  произвольные hex-значения в JSX.

## Что НЕ делать
- Не добавлять состояние в localStorage/sessionStorage помимо выбранного языка.
- Не тянуть тяжёлые UI-библиотеки (MUI, AntD и т.п.) — весь UI кастомный на Tailwind.
- Не создавать отдельные страницы/роутинг — сайт остаётся одностраничным с якорями.
- Не хардкодить количество кейсов — секция портфолио должна рендериться из `data/projects.js`
  через `.map()`, добавление кейса = добавление объекта в массив.
